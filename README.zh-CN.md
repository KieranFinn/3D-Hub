# 3D Hub

[English](./README.md) | **简体中文**

> 一个由 AI 驱动的 3D 角色创作工作室，用于生成、微调并导出 3D 人物形象。

## 项目概述

3D Hub 是一个 **自然语言驱动的 3D 角色创建项目**。

它不希望用户一上来就面对一堆底层滑杆参数，而是从 **意图** 出发：
用户先用自然语言描述自己想要的人物，系统再理解这段描述，生成结构化的人物配置，渲染出 3D 预览，并允许用户通过交互式控制台继续微调细节。

这个项目的长期目标，不只是做一个网页里的可调头像，而是建立一条完整链路：

**文字描述 → 结构化角色定义 → 可编辑的 3D 角色 → 可导出的 3D 模型**

最终，导出的模型应支持以下使用场景：
- 资产复用
- 视觉展示
- 二次编辑
- 3D 打印

---

## 产品愿景

3D Hub 希望成为一个角色创作平台，让用户能够：

1. 用自然语言**描述**心中的人物
2. 从描述中**生成**可视化的 3D 角色
3. 通过预设和参数继续**微调**角色
4. **保存和导出**最终角色配置与模型
5. 为未来的制造或 3D 打印流程做好准备

一句话概括：

> 3D Hub 把想象中的人物，变成可编辑、可导出的 3D 角色。

---

## 当前方向

当前原型的重点，是先建立一种 **舞台优先（stage-first）** 的角色创作体验：

- 中央是 3D 角色主舞台
- 控制台以悬浮方式叠加在舞台上
- 自然语言输入作为主要创建入口
- 预设和精细调节作为二次优化工具

这个方向有意让产品更接近 **角色实验室 / 创作工作室**，而不是传统的网页表单或后台控制台。

---

## 核心产品流程

1. 用户输入角色描述
2. 系统把文本解析成结构化角色参数
3. 将角色状态应用到 3D 模型
4. 用户通过预设和参数控件继续微调
5. 用户导出配置，后续再导出完整 3D 模型

---

## MVP 范围

第一阶段的实际目标，是先验证这条最小闭环：

**自然语言输入 → 角色状态 → 3D 预览 → 手动微调 → 配置导出**

### MVP 目标
- 在浏览器中渲染一个 3D 角色
- 支持一小组可编辑的面部参数
- 增加自然语言提示输入区
- 支持风格预设
- 支持保存 / 导出当前角色配置

### 第一版暂不覆盖
- 全自动高质量可打印网格生成
- 完整的工业级角色资产生产管线
- 完整账号系统
- 高级多人协作能力

## 推荐架构

原型正在从“单文件 demo”逐步迁移到更清晰的分层结构。

### 推荐目录结构

```text
3dhub/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── main.js
│   ├── state.js
│   ├── renderer.js
│   ├── character.js
│   ├── controls.js
│   ├── presets.js
│   └── text-parser.js
├── assets/
│   └── models/
├── data/
│   └── presets.json
└── docs/
    └── ARCHITECTURE.md
```

### 各文件职责

- `state.js` —— 角色参数的唯一数据源
- `character.js` —— 角色网格创建与更新逻辑
- `renderer.js` —— Three.js 场景、相机、灯光、渲染器
- `controls.js` —— UI 事件与状态更新
- `presets.js` —— 风格预设模板
- `text-parser.js` —— 未来的文本转角色配置层
- `main.js` —— 应用初始化与模块粘合层

## UI / 布局方向

目标布局 **不是** 传统的左右后台分栏。

3D Hub 更应该像一个 **3D 角色创作工作台**：

- 全屏舞台背景
- 中央角色展示区
- 舞台右侧悬浮控制台
- 叠加少量轻信息卡片
- 页面具有明确的层次、深度和空间构图

### 视觉关键词
- 赛博朋克灵感
- 未来感
- 舞台感
- 分层
- 霓虹点缀但不过度
- 更像工具，而不是后台

---

## 设计原则

### 1. 角色优先
角色和舞台必须是体验的视觉中心。

### 2. 自然语言优先
滑杆只是微调工具，不应成为主要入口。

### 3. 参数驱动架构
UI 应该更新统一的角色状态，而不是到处直接修改场景对象。

### 4. 从一开始就考虑导出
系统架构应提前为未来导出链路预留能力。

### 5. 产品感优先于 demo 感
体验应该像一个创作工作室，而不是技术验证页。
